# 基于规则和词库的文本生成器

一个基于 Java Swing 的桌面应用程序，支持通过自定义词库和句子模板规则，自动生成符合语法结构的中文句子。

## 📋 项目特性

- **词库管理**：支持词语的增删改查，按词性（名词、动词、形容词等）和分类（自然、科技、情感等）组织
- **模板引擎**：自定义句子结构规则，如 `[分类:自然][词性:形容词] + [分类:自然][词性:名词]`
- **随机生成**：根据模板规则从词库中智能匹配并随机抽取词语
- **持久化存储**：词库和模板均保存为本地 JSON 文件
- **图形界面**：直观的 Swing GUI，包含词库管理、模板编辑、句子生成三个功能页面

---

## 🏗️ 类结构设计

| 类名 | 包路径 | 职责说明 |
|------|--------|----------|
| `Word` | `com.textgenerator.model` | 词语数据模型，包含 text(文本)、pos(词性)、category(分类) 属性 |
| `WordLibrary` | `com.textgenerator.library` | 词库管理器，负责加载/保存 JSON 文件、添加/删除词语、按条件筛选词语 |
| `TemplateParser` | `com.textgenerator.parser` | 模板解析器，将模板字符串解析为规则列表，支持 `[分类:xxx][词性:yyy]` 格式 |
| `TextGenerator` | `com.textgenerator.engine` | 生成引擎，接收模板和词库，随机匹配词语并拼接成完整句子 |
| `MainGUI` | `com.textgenerator.gui` | 主界面，使用 JTabbedPane 组织三个功能面板 |

## 🚀 快速开始

### 环境要求

- JDK 17 或更高版本
- Java Swing (内置于 JDK)
- JSON 库 (已包含在 `resources/json.jar`)

### 编译项目

```bash
cd /workspace

# 创建输出目录
mkdir -p out

# 编译所有源文件
javac -d out -cp "resources/json.jar" \
    src/com/textgenerator/model/Word.java \
    src/com/textgenerator/library/WordLibrary.java \
    src/com/textgenerator/parser/TemplateParser.java \
    src/com/textgenerator/engine/TextGenerator.java \
    src/com/textgenerator/gui/MainGUI.java \
    src/com/textgenerator/TextGeneratorTest.java
```

### 运行方式

#### 方式一：运行 GUI 界面（需要图形显示环境）

```bash
java -cp "out:resources/json.jar" com.textgenerator.gui.MainGUI
```

#### 方式二：运行命令行测试（无 GUI 环境推荐）

```bash
java -cp "out:resources/json.jar" com.textgenerator.TextGeneratorTest
```

## 📝 数据文件格式

### 词库文件 (`resources/word_library.json`)

```json
[
  {
    "text": "青山",
    "pos": "名词",
    "category": "自然"
  },
  {
    "text": "蔚蓝",
    "pos": "形容词",
    "category": "自然"
  },
  {
    "text": "奔跑",
    "pos": "动词",
    "category": "动作"
  },
  {
    "text": "快速",
    "pos": "副词",
    "category": "通用"
  }
]
```

**字段说明：**
- `text`: 词语本身
- `pos`: 词性（名词、动词、形容词、副词等）
- `category`: 分类（自然、科技、情感、动作、通用等）

### 模板文件 (`resources/templates.json`)

```json
{
  "templates": [
    {
      "name": "自然描写",
      "pattern": "[分类:自然][词性:形容词] + [分类:自然][词性:名词]"
    },
    {
      "name": "简单句",
      "pattern": "[词性:名词] + [词性:动词] + [词性:名词]"
    },
    {
      "name": "复杂句",
      "pattern": "[分类:自然][词性:形容词] + [分类:自然][词性:名词] + [词性:动词] + [词性:副词] + [词性:动词]"
    }
  ]
}
```

**模板语法：**
- `[分类:xxx]`: 限定词语必须属于指定分类
- `[词性:yyy]`: 限定词语必须是指定词性
- `+`: 连接符，表示词语之间的分隔（可自定义）
- 分类和词性可以组合使用，也可以单独使用

## 🎯 使用指南

### 1. 词库管理页

- **查看词库**: 表格展示所有词语，支持按列排序
- **添加词语**: 点击"添加词语"按钮，输入文本、选择词性和分类
- **删除词语**: 选中表格中的行，点击"删除选中"按钮
- **保存修改**: 点击"保存到文件"按钮，将更改写入 JSON 文件
- **刷新词库**: 点击"重新加载"按钮，从文件重新读取词库

### 2. 模板管理页

- **查看模板**: 下拉框显示所有已保存的模板名称
- **编辑模板**: 在文本域中修改当前模板的规则
- **保存模板**: 点击"保存模板"按钮，更新模板文件
- **新建模板**: 输入新名称和规则，点击保存即可添加

### 3. 句子生成页

- **选择分类**: 从下拉框选择词语的分类范围（可选"全部"）
- **选择模板**: 从下拉框选择预设的句子模板
- **生成句子**: 点击"生成句子"按钮，随机生成符合规则的句子
- **查看结果**: 生成的句子显示在只读文本区
- **复制结果**: 双击结果文本可全选复制

## 🔧 核心逻辑说明

### 模板解析流程

1. **正则匹配**: 使用正则表达式 `\\[([^\\]]+)\\]` 提取所有方括号内的规则片段
2. **规则拆分**: 将每个片段按冒号拆分为约束类型（分类/词性）和约束值
3. **构建规则对象**: 将解析结果封装为 `TemplateRule` 对象列表
4. **验证规则**: 检查规则的合法性，抛出异常提示错误格式

### 词语抽取流程

1. **遍历规则**: 对模板中的每个规则依次处理
2. **筛选词库**: 根据规则的分类和词性约束，过滤出候选词语列表
3. **随机选择**: 从候选列表中随机选择一个词语
4. **拼接结果**: 将所有选中的词语用连接符拼接成完整句子
5. **异常处理**: 如果某个规则没有匹配的词语，抛出异常提示

## ⚠️ 注意事项

1. **首次运行**: 程序会自动创建默认的词库和模板文件（如果不存在）
2. **文件编码**: 所有 JSON 文件使用 UTF-8 编码，确保中文正常显示
3. **词库完整性**: 删除词语前请确认，保存后无法撤销
4. **模板格式**: 模板必须严格遵循 `[类型:值]` 格式，否则解析失败
5. **随机性**: 每次生成结果可能不同，这是预期行为
